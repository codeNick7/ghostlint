## Goal
Add all 14 requested MCP tools to the ghostlint MCP server, reusing existing engine primitives (Scanner, ScanRecord cache, GitMetrics, GitAnalyzer, recommendations). Zero regression on existing 5 tools and their tests. Keep test infra current (unit + integration coverage for every new tool, plus close the existing `check_diff` test gap).

## Guiding principles
1. **No AI in the analysis loop** (per README): `search_repository_knowledge` = deterministic keyword/substring search over findings + files. No embeddings/network.
2. **Cache-first with auto-scan-on-miss**: focused tools (`find_*`, cleanup, metrics) read the latest persisted `ScanRecord`; if none exists they run+persist a fresh scan. A `force_refresh` flag re-scans. This reuses existing infra and matches how `get_health_context`/`list_findings` read the DB.
3. **Thin tools, pure helpers**: all "intelligence" logic goes in a new pure-function module `ghostlint_mcp/repo_intel.py` (mirrors the existing `context_builder.py` split). Tools in `server.py` are thin `@mcp.tool()` wrappers: resolve path → get scan/scan-result → call repo_intel → return dict (wrapped in `try/except` → `{"error": ...}`, matching the existing convention).
4. **Additive only**: no edits to detectors, scanner, health_score, recommendations, context_builder, or the 5 existing tools' behavior.

## New module: `app/ghostlint_mcp/repo_intel.py` (pure functions, unit-testable)
- `health_label(score) -> str` — "healthy"/"needs review"/"has significant issues"/"critical".
- `finding_to_dict(f) -> dict` — full finding shape incl. category/title/description/file/line/confidence/risk/effort/benefit.
- `overview(repo_path: Path) -> dict` — filesystem-only (no scan): files_by_language, total_files, total_lines, top_directories, detected_frameworks (from pyproject.toml/package.json/requirements.txt), entry_points (main.py/app.py/index.js/...), config_files. Uses `FileIndexer`.
- `cleanup_recommendations(findings) -> list[dict]` — sorted "quick wins first" (confidence desc, risk asc, effort asc).
- `estimate_effort(findings) -> dict` — maps `EffortLevel`→hours (minutes=0.5, hours=2, days=8), weighted by confidence; totals + by_effort + by_category + quick_wins_count.
- `build_cleanup_plan(findings, health_score) -> dict` — groups into 4 phases (Quick wins / Quick fixes / Larger refactors / Strategic) with per-phase items, counts, est_hours; plus recommended_order + summary.
- `metrics_dict(result, git_metrics) -> dict` — health score + per-category scores + git metrics + complexity signals.
- `patterns(findings, repo_path) -> dict` — synthesizes "repository patterns" from existing detectors: duplication patterns (duplicate_logic groupings), naming patterns (naming findings), api-proliferation patterns (refactor synonyms), plus directory-structure patterns from the filesystem. Deterministic, no new detector.
- `explain_history(repo_path: Path) -> dict` — git narrative via GitPython/git_metrics: recent commits, age, contributors, stability/velocity/friction, prose narrative. Best-effort (returns `available=False` if not a git repo).
- `search_knowledge(findings, repo_path, query) -> dict` — deterministic search: finding_matches (title/description/category/file substring + word-overlap score) + file_matches (file names/paths matching query). Ranked by score.
- `scan_timeline(records) -> dict` — from ScanRecord list: per-scan entry + trend (improving/declining/stable from first→latest score).

## Shared helpers added to `app/ghostlint_mcp/server.py`
- `_latest_record(repo_path) -> ScanRecord | None` — most recent persisted scan for the repo (desc by started_at).
- `_reconstruct_scan(record) -> ScanResult` — rebuild a `ScanResult` (HealthScore from health_score_json, Finding[] from FindingRecords with Evidence, recommendations via `generate_recommendations`). Lets cache-backed tools avoid rescans. (git_metrics are not stored in DB → tools needing them compute live via `compute_git_metrics`.)
- `_get_scan_result(repo_path, force_refresh=False, engines=None, min_confidence=0.6) -> ScanResult` — cache-or-scan: cache hit → `_reconstruct_scan`; miss/force → `_run_scan` (persisted). Raises on missing path (tools return `{"error": ...}`).
- `_find_by_category(repo_path, category, min_confidence, limit, force_refresh) -> list[Finding]` — shared by the 4 `find_*` tools.
